     ## 설계 전략 최종 정리
     
     ### 1. 에러 핸들링 4계층 구조
     
     ```
     [요청]
       → Controller (@Validated) — Bean Validation 자동 처리
       → Service — BusinessException / ExternalApiException throw
       → GlobalExceptionHandler — 모든 예외 중앙 포착
       → ErrorResponse — 통일된 JSON 반환
     ```
     
     **표준 응답 규격 (성공/실패 모두 동일 구조)**
     ```json
     // 성공
     { "success": true, "data": {...}, "message": null, "timestamp": "..." }
     
     // 실패
     { "success": false, "code": "X001", "message": "외부 서비스 불가", "errors": [], "timestamp": "..." }
     ```
     
     **핸들링 대상별 전략**
     
     | 상황 | 처리 방식 | HTTP 코드 |
     |---|---|---|
     | `@Validated` 실패 | `MethodArgumentNotValidException` → errors[] 배열 포함 | 400 |
     | 비즈니스 예외 | `BusinessException(ErrorCode.XXX)` | ErrorCode 정의값 |
     | 외부 API 다운 | CircuitBreaker OPEN → `ExternalApiException` | 503 |
     | 외부 API 타임아웃 | WebClient `TimeoutException` → fallback | 504 |
     | JWT 만료 | `ExpiredJwtException` → 필터에서 포착 | 401 |
     | 접근 권한 없음 | Spring Security `AccessDeniedException` | 403 |
     
     ---
     
     ### 2. 외부 API 장애 격리 — 3중 방어
     
     ```
     WebClient
       └─ connectionTimeout: 3초  (TCP 연결 실패 감지)
       └─ readTimeout: 10초       (응답 지연 감지)
           │
           ↓
     Resilience4j CircuitBreaker
       └─ 최근 5회 중 50% 이상 실패 → OPEN (30초 차단)
       └─ OPEN 중 → 외부 호출 없이 즉시 fallback
       └─ HALF_OPEN → 1회 시도 후 복구 여부 판단
           │
           ↓
     Fallback
       └─ ExternalApiException(EXTERNAL_API_UNAVAILABLE)
       └─ GlobalExceptionHandler → 503 + retryAfter: 30 응답
     ```
     
     ---
     
     ### 3. 퇴사자 즉시 차단 — 사용자 제안 + 개선안 비교
     
     **사용자 제안:** Filter 레벨에서 DB 조회 → 상태 체크
     
     > 올바른 방향입니다. 다만 단독으로는 **이미 발급된 JWT가 남아있으면 만료 전까지 재사용 가능**한 취약점이 있습니다.
     
     **내가 추천하는 이중 잠금:**
     
     ```
     [퇴사 처리 API 호출]
       → status = RESIGNED
       → tokenVersion++ (예: 0 → 1)
       → DB 저장
     
     [다음 요청 시 JwtAuthenticationFilter]
       → JWT 파싱 → claim에서 tokenVersion 추출 (발급 시점: 0)
       → DB에서 현재 tokenVersion 조회 (현재: 1)
       → 0 ≠ 1 → 즉시 401 반환 (토큰 유효기간 남아있어도 차단)
     
     [CustomUserDetailsService]
       → status == RESIGNED → isEnabled() = false
       → Spring Security DisabledException → 403
     ```
     
     **두 레이어 역할:**
     - `tokenVersion`: 기존 발급 토큰 즉시 무효화 (퇴사 처리 순간부터)
     - `isEnabled()`: 신규 로그인 자체를 Spring Security 표준으로 차단
     
     추가 인프라(Redis 등) 없이, DB 컬럼 1개(`token_version INT DEFAULT 0`)로 구현 가능합니다.
